Duplicate Case Visits
======================================

.. include:: ../../common/stub-overview.txt

During the medical camp, a field user creates a new visit and conduct survey's for every individual visiting the camp.  
A duplicate case is created when:

1. Individuals re-visit camps in different locations or
2. Common data is shared between individuals. 

For instance, the phone number shared among the members of a family.

.. Note:: *Application can recognize duplicate data, if de-duplication happens for fields - Mobile number, Aadhaar no and date of birth of individuals.*

Admin can de-duplicate case by:

1. `Associate Case`_ or
2. `Create New Case`_

Possible Duplicates 
------------------------------ 
To view possible duplicates:

* Select a Study on the Home page and click Possible Duplicates tab on the Summary. OR
* Goto Manage, Visits drop-down and click Possible Dupicates. 
* On the list of studies with duplicate data, select a visit. Duplicate case for the current study will be listed on the right. 

You can either associate it to a case or create a new case. 

.. _Associate Case

Associate Case
-------------------------- 
To associate the case to the current study:

* Select a visit from the  possible list of duplicate cases.
* On the right, scroll and select a duplicate case from the list and click Associate Case.  

  .. image:: images/possibleduplicates.jpg

The case will be associated to the current study and removed from the list of possible duplicates.

.. _Create new Case

Create New Case
-------------------------------------
If you do not want to associate the visit to an existing case, then a new case can be created.  

* Select a visit from the possible duplicate list of data.
* Scroll-over to Possible Duplicate section on the right, and click Create New Case. A message pop-up appears to confirm new case has been created.

A new case with the study data will be created and listed under **Manage**, **Cases** under the **Home** page.
